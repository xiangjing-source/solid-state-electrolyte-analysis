# 核心教训与注意事项

## ⚠️ 科学参数必须来自数据源，禁止 LLM 编造

**规则**：涉及材料科学计算的输入参数（空间群、晶胞参数、组成等），必须从 CIF 文件、数据库 CSV、Materials Project API 等结构化数据源提取。绝对不能使用 LLM "知识回忆"来填充这些数值。

**反面案例**：LGPS 预测时，LLM 编造了空间群 No.141（实际 137）和晶胞参数 a=8.9715（实际 8.719），导致预测结果偏差 2 个数量级。

**正确做法**：查描述符从 OBELiX 全量（train+test）→ 再查 Materials Project API → 最后才是 LLM 兜底（仅用于非数值型字段）。真实实验值也从 OBELiX 全量查，有就报误差，没有就不写。

**禁止**：LLM 编造空间群、晶胞参数等数值型数据。

## ⚠️ mp_id 必须来自真实查询，禁止编造

**规则**：报告末尾的 MP 链接必须基于 MP API 真实查询返回的 material_id 拼接，不能凭空编造 mp-id。

**反面案例**：之前输出了不存在的 mp-23222 链接，用户发现后指出。

**正确做法**：每个化学式都必须调 MP API 查 mp_id，没有则说明"MP 暂无收录"。

## ⚠️ CIF 文件参数可能与数据库不一致

**规则**：用户提供的 CIF 文件空间群和晶胞参数可能与 OBELiX、MP 的记录不同（如 Li₇BiO₆：CIF=P1, OBELiX=P-1, MP=R3）。按规则以 CIF 提供的参数跑模型，但在报告中标注差异。

**影响**：参数不一致会导致预测误差偏大，需在报告中诚实说明。

## ⚠️ unseen 材料预测置信度标注

**规则**：材料不在 OBELiX 中（unseen）时，必须在报告中明确标注预测置信度低。尤其当训练集缺少相关元素（如 H、N）时，应说明是纯外推，数值仅供参考。

**反面案例**：Li₂Mg(NH)₂ 含 H 和 N，训练集 599 条中无任何含 H/N 样本，预测值 ~10⁻⁹ S/cm 不可信。

## ⚠️ mp/ 目录是测试数据，禁止用于数据源查询

**规则**：mp/all.csv、mp/seen_materials.csv、mp/unseen_materials.csv 等文件是用户用来做模型测试的，绝对不能作为数据源查询的依据。

**正确做法**：数据源查询用 OBELiX 全量（train+test），mp/ 目录仅用于独立的模型评估对比。

## ⚠️ MP 中不存在的材料

以下材料在 Materials Project 中没有收录：
- Li₂ZrCl6（Li-Zr-Cl 三元体系整个不在 MP）
- Li₉N₂Cl3（Li-N-Cl 体系也不在）
